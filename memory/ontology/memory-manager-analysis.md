## 一、技能概述

**Memory Manager** 实现了 **语义/程序/情节三层记忆模式**（Semantic/Procedural/Episodic），是专业级 AI Agent 记忆架构工具。

## 二、三层记忆架构详解

### 2.1 三层对比

| 层级 | 存储路径 | 查询方式 | 示例 |
|------|----------|----------|------|
| **Episodic** (发生了什么) | `memory/episodic/YYYY-MM-DD.md` | "我上周二做了什么？" | 每日事件日志 |
| **Semantic** (我知道什么) | `memory/semantic/topic.md` | "关于 X 我知道什么？" | 主题知识库 |
| **Procedural** (怎么做) | `memory/procedural/process.md` | "如何执行 Y 流程？" | 工作流文档 |

### 2.2 核心能力 (10项)

| 能力 | 自动化程度 | 说明 |
|------|-----------|------|
| Episodic 存储 | 手动 | 时间顺序事件日志 |
| Semantic 存储 | 手动 | 事实/概念/知识沉淀 |
| Procedural 存储 | 手动 | 工作流/模式/流程文档 |
| 压缩检测 | 全自动 | 监控所有记忆类型使用率 |
| 快照保存 | 半自动 | 压缩前保存完整记忆状态 |
| 记忆组织 | 手动 | 迁移 flat files 到三层结构 |
| 类型搜索 | 全自动 | episodic/semantic/procedural/all 四种搜索模式 |
| 统计报告 | 全自动 | 各类型条目数量/大小/增长率 |
| 手动分类 | 手动 | categorize.sh 辅助手动分类 |
| 历史迁移 | 手动 | 将 legacy flat files 迁移到三层结构 |

## 三、已知限制与边界 (7项)

| 边界 | 类型 | 严重度 | 说明 |
|------|------|--------|------|
| Shell 脚本依赖 | 依赖 | 🔴 **严重** | 所有功能依赖 Bash，Windows 原生环境可能无法运行 |
| 手动分类为主 | 策略 | 🟡 警告 | 自动化程度低，无 ML 自动分类能力 |
| 无向量搜索 | 技术 | ⚪ 信息 | 仅关键词搜索，与 LanceDB 向量记忆互补 |
| 搜索精度有限 | 技术 | ⚪ 信息 | 基于 grep，无法理解语义关系 |
| 无知识图谱 | 技术 | ⚪ 信息 | v1.1 路线图承诺但未实现 |
| 无实时压缩预测 | 技术 | ⚪ 信息 | 仅被动检测，无法主动预测 |
| 单 Agent 记忆隔离 | 技术 | 🟡 警告 | 多 Agent 无法共享记忆 |

## 七、与 Elite Longterm Memory 的互补关系

**关键差异：**

| 维度 | Memory Manager | Elite Longterm Memory |
|------|---------------|----------------------|
| 架构 | 3-tier（文件组织） | 6-layer（分层存储） |
| 核心能力 | 压缩检测 + 组织工具 | WAL 协议 + 向量搜索 + 云备份 |
| 自动化程度 | 低（手动为主） | 高（WAL 自动捕获） |
| 搜索能力 | 关键词搜索 | 语义向量搜索 |
| 适用场景 | 记忆分类和压缩保护 | 跨会话记忆持久化 |

**推荐组合使用：**
- **Elite Memory** 作为主要的写入协议（Hot RAM/WAL）
- **Memory Manager** 作为分类组织工具（episodic/semantic/procedural）
- **LanceDB** 提供语义向量搜索能力
- **Ontology** 提供实体关系图谱
