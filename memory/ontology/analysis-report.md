## 三、三技能联动架构

```
                    ┌─────────────────────┐
                    │    Baidu Search      │
                    │  (信息获取层)         │
                    └──────┬──────┬───────┘
                           │      │
              搜索辅助审查   │      │  搜索结果图谱化
              (Pipeline)    │      │  (Pipeline)
                           ▼      ▼
    ┌──────────────┐              ┌──────────────┐
    │ Skill-Vetter │◄────────────►│   Ontology   │
    │ (安全守门层)  │  审查↔记录    │ (知识存储层)  │
    └──────────────┘  (双向)       └──────────────┘
```

### 联动模式详解

| # | 联动路径 | 模式 | 说明 |
|---|----------|------|------|
| 1 | Search → Vetter | Pipeline | 搜索技能作者/仓库背景，增强来源检查 |
| 2 | Search → Ontology | Pipeline | 重要搜索结果存为Document实体，积累知识 |
| 3 | Vetter → Search | Callback | 审查时搜索技能安全相关信息 |
| 4 | Vetter → Ontology | Pipeline | 审查报告存入图谱，形成安全档案库 |
| 5 | Ontology → Vetter | Shared State | 查询已审查记录，决定是否重新审查 |
| 6 | Ontology → Search | Event Driven | 发现知识缺口时自动触发搜索补充 |

### 推荐完整工作流

```
用户请求安装技能
    │
    ▼
① find-skills 搜索候选
    │
    ▼
② baidu-search 搜索作者/技能背景信息
    │
    ▼
③ skill-vetter 安全审查 (结合搜索结果)
    │
    ├─ ✅ SAFE → 安装 → ontology 记录技能+审查结果
    ├─ ⚠️ CAUTION → 提示人工确认
    └─ ❌ REJECT → ontology 记录拒绝原因
    │
    ▼
④ ontology 更新图谱，下次可查询历史
```

## 四、最佳实践汇总 (10项)

### 安全类
1. **安装前必审** — 任何来源的技能安装前都先经 skill-vetter 审查
2. **最小权限原则** — 重点关注技能是否请求了超出功能所需的权限
3. **审查前先搜索** — 用 baidu-search 搜索作者和技能背景，增强审查质量
4. **定期复审** — 技能版本更新后重新审查

### 工作流类
5. **审查记录留档** — 每次审查结果写入 ontology 图谱
6. **搜索结果入图** — 重要搜索结果存入 ontology 作为 Document 实体

### 架构类
7. **Schema 优先** — 使用 ontology 前先定义类型约束
8. **追加不覆盖** — ontology 数据变更始终 append/merge
9. **技能契约声明** — 使用 ontology 的技能应声明 reads/writes

### 性能类
10. **善用时间过滤** — baidu-search 搜索时事用 freshness 参数提高相关性
