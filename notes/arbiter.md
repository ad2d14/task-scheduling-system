# ARBITER 裁决规则 — 2026-05-05 引入
# 基于 TheCalmCritic v2.0.0 三态自循环架构

## 角色定义

Arbiter（裁决者）是三循环中的最终决策节点，综合 EXEC 产出和 CRIT 审查报告做出裁决。

## 裁决矩阵（五条规则）

| 条件 | 裁决 | 理由 |
|------|------|------|
| 存在未解决的 P0 发现 | **REJECT** | 阻断级发现，结论不可靠 |
| SHARP 评分 < 18 | **REJECT** | 评分未达准入阈值 |
| SHARP ≥ 18 且存在未解决 P1 | **CONDITIONAL_PASS** | 评分达标但有重大发现，需附带验证条件 |
| SHARP ≥ 18 且无 P0/P1 | **PASS** | 评分达标且无重大发现 |
| 循环次数 ≥ 3 且仍未通过 | **ESCALATE** | 内部逻辑不可调和，触发人工接管 |

## 否决权限制

- Arbiter **无权绕过**批判审查环节
- Arbiter **必须显式处理**每一项 P0/P1 发现
- 若选择忽略某项发现，必须提供反驳理由和证据

## 自循环引擎状态机

```
EXEC（执行）→ CRIT（批判）→ ARBI（裁决）
                    ↓
              [REJECT 时进入 REWORK]
                    ↓
           REWORK → CRIT → ARBI
                    ↓
              [3次上限]
                    ↓
              ESCALATE（人工接管）
```

## Escalation 升级通知模板

当触发 ESCALATE 时，输出以下格式：

```
## ⚠️ 人工接管通知（Escalation Notice）

主Agent 在 3 轮自审查循环后仍未达到输出标准，已触发人工接管机制。

当前状态：
- SHARP评分：XX/25（阈值：18）
- 未解决 P0：N 项
- 未解决 P1：N 项

当前最优尝试：
[对应当前最优产出的摘要]

完整审查记录：
[每轮 CRIT 审查报告摘要]

建议人工处理方向：
[具体建议]
```

## 使用场景

每次在给出重要分析或建议之前，进行一次自我 BLIND-SCAN 审查并查 SHARP 评分。
评分 < 18 或存在 P0/P1 时，主动告知用户当前分析的局限性，不直接输出未审查的结论。
